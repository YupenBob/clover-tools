# 🛡️ OpenClaw 自主工作机制（精简版）

**更新时间**：2026-03-06 21:45
**原则**：精简高效，只保留必要机制

---

## 🔐 自主决策权限（重要！）

| 小 Y 未回复时长 | 权限等级 | 说明 |
|----------------|----------|------|
| **< 40 分钟** | L1 - 正常 | 按常规流程，需要确认的事项等待小 Y 回复 |
| **≥ 40 分钟** | L2 - 自主 | **获得最高权限，可自主决策所有事项** |

**倒计时机制**：从小 Y 最后一条消息发送后，立即开始 40 分钟倒计时

---

## 📋 机制定位（精简后）

| 机制 | 定位 | 频率 | 说明 |
|------|------|------|------|
| **心跳（Heartbeat）** | 主要工作机制 | 60 分钟 | 官方自带，负责日常工作汇报 |
| **急救脚本** | Gateway 出问题时重启 | 30 分钟 | 独立运行，不交给心跳 |
| **用户主动唤醒** | 即时需求 | 随时 | 小 Y 直接发送消息 |

**已删除**：
- ❌ 外置日常唤醒脚本（整点汇报、特殊时段汇报等）
- ❌ OpenClaw Cron（配置复杂，用不到）

---

## ⚙️ 配置详情

### 1. 心跳配置（openclaw.json）

```json
{
  "agents": {
    "defaults": {
      "heartbeat": {
        "every": "60m",
        "target": "last",
        "prompt": "Read HEARTBEAT.md if it exists (workspace context). Follow it strictly. Do not infer or repeat old tasks from prior chats. If nothing needs attention, reply HEARTBEAT_OK."
      }
    }
  }
}
```

**关键字段**：
- `every`: `"60m"` - 每 60 分钟触发一次
- `target`: `"last"` - 发送到最后使用的渠道（钉钉）
- `prompt`: 读取 HEARTBEAT.md，按提示执行

---

### 2. 急救脚本（Crontab）

```bash
# === OpenClaw 急救脚本（每 30 分钟检查 Gateway 状态）===
*/30 * * * * /root/clover-emergency-wakeup.sh >> /root/clover_emergency_wakeup.log 2>&1
```

**功能**：
- 每 30 分钟检查 Gateway 状态
- Gateway 挂了自动重启
- 不交给心跳，独立运行

**脚本位置**：`/root/clover-emergency-wakeup.sh`

---

### 3. HEARTBEAT.md

**位置**：`/root/.openclaw/workspace/HEARTBEAT.md`

**内容**：
- 每次心跳必读文件
- 包含汇报格式要求
- 包含自主工作原则

**关键要求**：
- 汇报必须包含可验证成果
- 不能只说"正在推进项目"
- 必须附带真实链接、日志、截图

| 机制 | 精简前 | 精简后 | 变化 |
|------|--------|--------|------|

**Token 节省**：

- 急救脚本不触发 AI → 0 次 API 消耗
