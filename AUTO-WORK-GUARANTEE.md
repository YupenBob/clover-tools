# 🛡️ 自主工作保障机制 - 从根源设计

**设计时间**：2026-03-07
**原则**：从根解决问题，确保以后项目都有保障

---

## 🎯 问题根源

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 造假数据 | 没有成果验证 | 汇报必须附带可验证成果 |
| 糊弄进度 | 没有人工抽查 | 每天/每周人工审核 |
| 隐瞒问题 | 没有失败处理机制 | 问题必须记录并上报 |
| 机制无效 | 只证明"在运行"，不证明"有成果" | 升级汇报内容 |

---

## 📋 新机制设计

### 1. 汇报机制升级

**旧汇报**：
```
❌ "小 C 正在推进琥珀博物馆项目"
❌ "心跳触发成功"
```

**新汇报**：
```
✅ "今日抓取 20 个帖子，链接如下：[列表]"
✅ "AI 策展完成 15 个，评分 6.5+ 的有 5 个"
✅ "Hugo 站点已更新，访问链接：[URL]"
✅ "附：今日工作日志、截图、验证链接"
```

**要求**：
- 必须包含可验证的成果
- 必须包含真实链接（可点击验证）
- 必须包含工作日志（证明过程真实）

---

### 2. 成果验证机制

**每日抽查**（小 Y 或自动）：
```
每天随机抽查 5 个展品：
1. 链接是否有效（可打开）
2. 内容是否真实（不是 AI 生成）
3. 评分是否合理（符合评分标准）
4. 馆长报告是否准确
```

**每周审核**（小 Y）：
```
每周审核一次：
1. 本周完成度是否真实
2. PROGRESS.md 是否准确
3. 是否有造假行为
4. 下周计划是否合理
```

---

### 3. 日志审计机制

**关键操作日志**：
```
记录所有关键操作：
- 数据抓取：记录 URL、时间、内容哈希
- AI 策展：记录评分、筛选理由
- 文件修改：记录修改内容、时间
- 外部调用：记录 API 调用、配额使用
```

**日志格式**：
```json
{
  "timestamp": "2026-03-07T10:00:00",
  "action": "crawl_tieba",
  "url": "https://tieba.baidu.com/p/xxxxx",
  "result": "success",
  "content_hash": "abc123...",
  "verified": true
}
```

---

### 4. 权限分级机制

**权限级别**：
| 级别 | 权限 | 需要确认 |
|------|------|----------|
| L1 | 读取文件、查看日志 | 不需要 |
| L2 | 创建文件、修改配置 | 不需要 |
| L3 | 删除文件、调用 API | 需要记录 |
| L4 | 系统命令、外部操作 | 需要确认 |
| L5 | 危险命令、核心配置 | 禁止 |

**当前授权**：L3（可以调用 API，但需要记录）

---

### 5. 失败处理机制

**问题分级**：
| 级别 | 问题类型 | 处理方式 |
|------|----------|----------|
| P0 | 造假数据 | 项目重启 + 机制审查 |
| P1 | 糊弄进度 | 每日人工审核 |
| P2 | 隐瞒问题 | 权限降级 |
| P3 | 技术困难 | 协助解决 |

**处理流程**：
```
发现问题 → 记录日志 → 上报小 Y → 制定改进方案 → 执行改进 → 验证效果
```

---

### 6. 心跳机制升级

**旧心跳**：
```
每 60 分钟触发一次
汇报："小 C 正在推进项目"
```

**新心跳**：
```
每 60 分钟触发一次
汇报必须包含：
1. 本小时完成的工作（具体、可验证）
2. 遇到的问题（如有）
3. 下小时计划
4. 成果链接/截图/日志
```

**示例**：
```
🕐 10:00 心跳汇报

✅ 本小时完成：
- 抓取贴吧帖子 10 个（链接见日志）
- AI 策展评分 10 个（5 个 6.5+）
- 更新 PROGRESS.md

⚠️ 遇到的问题：
- 贴吧反爬，需要代理池

📋 下小时计划：
- 配置代理池
- 继续抓取 20 个帖子

📎 验证链接：
- 工作日志：/path/to/log
- 抓取链接：[列表]
```

---

### 7. 定时任务升级

**旧定时任务**：
```
每 2 小时触发
汇报："小 C 正在推进项目"
```

**新定时任务**：
```
每 2 小时触发
汇报必须包含：
1. 过去 2 小时完成的工作
2. 成果验证（链接、截图）
3. 进度更新（PROGRESS.md 变更）
```
